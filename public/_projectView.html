Record Keeper

The purpose of the application is to allow users to keep track of their growing record collections. Users can also see what albums have been uploaded and check out other users music collections. It was a collaborative project built on Ruby on Rails Framework and library.

We utilized Postgresql for database mangagement and Sorcery gem for user authentication and sessions. My contribution was getting album forms to post to the database and ensuring users only had access to their uploaded content. I also as confirmed all routes functioned properly and routed to correct locations.

10 Mile Stereo

The purpose of the application is to allow users to connect with fellow music lovers. User's can post news, pictures, album reviews and follow other users that post interesting things. This site is built on Ruby on Rails with Bootstrap, and little bit of javascript.

The site has email authentication and utilizes Heroku sendgrid addon to send activation email to users. I used bycrypt for security and faker gem to seed database. Postgresql was used for database management. Site was deployed to Heroku with custom domain.

Guest Advisor

This application had a different take on services like tripadvisor. It allows hotel personnel to create accounts and share reviews about guests. It was a collaborative application built using Node.Js, Express and MongoDB. Passport was used to control user sessions. My contribution entailed creating Ajax calls to guest collection in our API for autocomplete search functionality and setting up database schema. I also was in charge of deploying application to Heroku and troubleshooted and resolved issues that arose during deployment.

Ticketbox

The purpose of this application is to be a concert ticket scrapbook. As someone who loves going to shows and holding on to tickets, I wanted some way to keep them all safe. I built this application from scratch using MeteorJs. I used MeteorJs because I want to learn about different frameworks and how they vary in structure and their best uses. The goal was for it be to a mobile application and when compiled into xcode allows access to user's camera and photo library to snap a picture of their tickets.
